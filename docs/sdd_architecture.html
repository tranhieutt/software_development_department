CORE CONFIGURATION

"Nguồn chân lý (Source of Truth) — chỉ định môi trường thực thi của toàn bộ vòng đời Agent."

Tệp cấu trúc cơ sở tiên quyết được phân phối vào bộ nhớ khung ngay khi khởi động Project. Nó định danh công nghệ kỹ thuật lõi mà toàn bộ hệ thống phải bám theo.

6 Ràng Buộc Tương Tác Bất Khả Xâm Phạm

  • NO AUTOPILOT: Làm rõ → Tùy chọn → Quyết → Nháp → Cấp phép.
  • NEVER WRITE DIRECTLY: Yêu cầu quyền Explicit access trước khi thao tác file I/O.
  • NO COMMITS: Buộc Batch-approval khi sửa đổi Multi-file changeset.
  • GIT SNAPSHOT: Bắt buộc chạy git status thực tế để quét thay đổi.
  • SAFETY TIERS: Định danh chỉ số Rủi ro (Low/Medium/High) trước lệnh Terminal.
  • ANNOTATION PROTOCOL: Ghi log sự cố ngầm vào annotations.md duy trì bối cảnh liên phiên.

Phân Cấp Tác Tử

AGENT TIERING

"Hệ thống điều phối chức năng chuyên trách, giảm thiểu rủi ro thiếu chính xác (Hallucination) từ mô hình chung."

TIER 1 (Leadership / Strategic Level)
CTO · Tech Director · Producer
>> Hoạch định kiến trúc, Chiến lược nền tảng

TIER 2 (Leads / Process Level)
Product Manager · Lead Programmer · QA Lead
>> Phân tích nguyên tắc PRD, Kiểm soát cổng (Quality Gates)

TIER 3 (Specialists / Implementation)
Frontend · Backend · Security · DevOps...
>> Cài đặt chi tiết mã nguồn, CI/CD Pipeline

Áp dụng Ma trận Leo thang (Escalation Matrix): Mọi rò rỉ hay lỗi tầng vi mô sẽ báo cáo lên cấp Lead, loại bỏ nguy cơ tác tử cấp thấp tự đưa phán quyết sai kiến trúc.

PART 04

Orchestration
Điều Phối Đa Nhiệm

MULTI-AGENT RELEGATION

"Khởi tạo giao thức chuyền gậy (Baton-passing) triệt tiêu tình trạng độc quyền phân tích mã đa diện."

Luồng thực thi tiêu biểu qua lệnh /team-feature authentication:

1. [Product Manager] Dịch nghĩa từ Requirement → Phân tích lập mốc PRD.
2. [UX Designer] Hoàn thiện bản đồ luồng User Flow / Wireframes.
3. [Technical Director] Đưa ra quyết nghị Kiến trúc (ADR) (Ví dụ: OAuth 2.0).
4. [Backend Dev] & [Frontend Dev] Khởi chạy song song mã nguồn gốc.
5. [Security Engineer] Đánh giá mã nguồn theo tiêu chuẩn mã độc (OWASP).
6. [QA Lead] Trích xuất Kiểm thử kịch bản Test Matrix.
(User giữ quyền quản trị và xác nhận tại mọi chốt chặn - Stage Gate)

PART 06

12 Điểm Kiểm Soát
Chốt Chặn CI/CD

EXECUTION CHECKPOINTS (HOOKS)

"Tích hợp hệ thống kiểm duyệt tĩnh (Continuous Inspection) trực tiếp tại vòng lặp của mô hình AI."

Các rào chắn kiểm thử chạy ngầm liên tục giám sát hệ sinh thái mã nguồn:

  • validate-commit.sh: Rà soát Hardcoded Secrets, định danh Syntax TODO bắt buộc.
  • validate-push.sh: Xác minh thông qua Unit Test Runner trước khi gửi dữ liệu lên Remote.
  • bash-guard.sh: Phân cấp rủi ro trước câu lệnh nguy hiểm (System removal, Network mapping).
  • detect-gaps.sh: Máy quét đo lường sai lệch chức năng giữa Codebase gốc và PRD thực tế.
  • pre-compact.sh: Backup nhanh Sandbox trước khi nén hay giải nén Context nội bộ.

PART 08

Fork-Join
Thực Thi Đồng Thời

PARALLEL EXECUTION

"Tạo môi trường ảo hóa đóng thùng (Containerized-like Environments) cho các nhánh AI Tác tử làm việc qua Git Worktrees."

Giải quyết triệt để nút thắt I/O cổ chai khi thao tác song song.
Process Flow: Agent lập trình cấu thành nhánh riêng, cách ly FORK làm việc độc lập → Trả về STATUS. Nhờ người thụ lệnh (User) ấn nút dung hợp mã với chiến lược JOIN (merge --no-ff) và thanh tẩy không gian ảo.

PART 09

Auto-Dream
Tối Ưu Hóa Bộ Nhớ

DATA CONSOLIDATION

"Tiến trình dọn dẹp trình nền định kỳ khi kết thúc phiên, hạn chế lãng phí tài nguyên (Token Load)."

Kích hoạt ngầm bởi session-stop.sh. Quy trình chia tách thành 5 giai đoạn:

  1. ORIENT: Đo đếm chỉ số Bytes, mốc dữ kiện Memory được đưa vào.
  2. DETECT: Soi chiếu các Logs thừa (>30 ngày / Cồng kềnh).
  3. ARCHIVE: Giáng cấp lưu trữ dư thừa vào Archive/Dreams.
  4. PRUNE: Bứng bỏ các liên kết Broken Markdown references khỏi mảng Index.
  5. LOG: Nắm bắt vào bản tin Trạng Thái phục vụ tra vết.
PART 10

Collaborative Protocol
Giao Thức Quản Trị

GOVERNANCE PROTOCOL

"User là người chủ động nắm quyền toàn phần trong Pipeline - AI là nhân sự đáp lại hiệu lệnh."

Quy chuẩn tương tác và bảo hành vòng lặp bất biến (Immutable State):

LÀM RÕ (Clarify) → HỎI ĐÁP ĐỊNH HƯỚNG (Options) → KỸ SƯ QUYẾT ĐỊNH (Decide)
→ NẠP BẢN NHÁP (Draft Proposal) → CẤP QUYỀN GHI NHẬN I/O (Approve & Write)

Rủi ro không bảo chứng nếu bỏ lỡ: Nếu Protocol thất lạc, mã nguồn đi kèm rủi ro Đè nén âm thầm (Silent Overwriting) gây hủy diệt hệ thống Architecture và rình rập tàn phát Database.
