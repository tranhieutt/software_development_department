PART 01

CLAUDE.md
Core Config

SYSTEM CONSTITUTION

"The foundational source of truth—defining the execution environment for the entire agent lifecycle."

The prerequisite configuration file injected into the context window upon session initialization. It establishes the core technical stack and operational parameters that the multi-agent system must strictly adhere to.

6 Immutable Technical Constraints

  • NO AUTOPILOT: Mandatory workflow: Clarify → Options → Decide → Draft → Approve.
  • NEVER WRITE DIRECTLY: Explicit authorization required prior to any I/O operations.
  • NO COMMITS: Batch-approval mandated for multi-file changesets.
  • GIT SNAPSHOT: Mandatory verification via git status prior to state mutations.
  • SAFETY TIERS: Risk classification (Low/Medium/High) required before terminal execution.
  • ANNOTATION PROTOCOL: Implicit documentation of API workarounds into annotations.md to preserve cross-session context.

PART 04

Orchestration
Multi-Agent Handoff

BATON-PASSING PROTOCOL

"Baton-passing protocols eliminating single-node analysis monopolies."

Standard execution sequence for /team-feature authentication:

1. [Product Manager] Translates requirements → Establishes the PRD.
2. [UX Designer] Finalizes user flow and UI wireframes.
3. [Technical Director] Issues Architecture Decision Record (ADR) (e.g., OAuth 2.0 vs JWT).
4. [Backend Dev] & [Frontend Dev] Initiate parallel non-blocking implementation.
5. [Security Engineer] Conducts vulnerability scans matching OWASP targets.
6. [QA Lead] Generates the comprehensive test matrix.
(Human operator retains stage-gate approval/override authority at every node)

PART 06

12 Hooks
CI/CD Checkpoints

AUTOMATED LIFECYCLE GATES

"Continuous inspection integrated directly into the AI's background execution loop."

Background validation scripts monitoring the repository ecosystem:

  • validate-commit.sh: Audits hardcoded secrets and enforces conventional TODO syntax.
  • validate-push.sh: Triggers unit test runners prior to any remote synchronization.
  • bash-guard.sh: Limits high-risk terminal commands (e.g., recursive removal, network mapping).
  • detect-gaps.sh: Measures functional drift separating actual implementation from the PRD.
  • pre-compact.sh: Generates state snapshots immediately prior to context mutation or compression.

PART 08

Fork-Join
Parallel Execution

ISOLATED WORKTREE BRANCHING

"Containerized-like virtualization enabling concurrent tasks via isolated Git Worktrees."

Alleviates underlying I/O bottlenecks during rapid parallel generation.

Process Flow: Associated agents branch into isolated execution directories (FORK) → operate completely autonomously → report STATUS metrics. The human operator ultimately triggers system integration (JOIN (--no-ff)) which merges code and prunes the virtualized environment without polluting the primary workspace architecture.

PART 09

Auto-Dream
Data Consolidation

BACKGROUND CONTEXT OPTIMIZATION

"Periodic background daemon clearing stale context to exponentially optimize token expenditure."

Triggered implicitly via session-stop.sh. Functions across 5 deterministic phases:

  1. ORIENT: Profiles aggregate byte volume and maps active memory nodes.
  2. DETECT: Flags stale references (>30 days untouched) or oversized architectural logs.
  3. ARCHIVE: Demotes redundant storage instances to the Archive/Dreams partition.
  4. PRUNE: Severs broken Markdown links from the primary indexing ledger.
  5. LOG: Serializes the garbage collection process for debugging and human auditability.
PART 10

Governance
Collaborative Protocol

PRESERVING HUMAN AUTHORITY

"The human operator retains unilateral terminal authority—the AI operates strictly as a responsive delegate."

Immutable interaction cycle ensuring code integrity and intention mapping:

CLARIFY (Ask) → PROVIDE OPTIONS (Analyze) → HUMAN DECIDES (Direct) 
→ DRAFT PROPOSAL (Model) → APPROVE & WRITE (Execute I/O)

Risk Mitigation Objective: Without this strict protocol, the codebase remains chronically vulnerable to silent overwriting and irrecoverable architectural hallucination. The system functions explicitly as a high-velocity consultancy engine, strictly bypassing the capacity of an autonomous decision-maker.
