Context

Problem statement

Without structure

An AI session has no department boundaries, no escalation path, no domain authority, and no memory across concerns. It answers every question but owns nothing. The result is organizational entropy: skipped design docs, silent technical debt, and scope creep without pushback.

SDD approach

A governed multi-agent harness — not a wrapper or prompt library. It imposes the coordination patterns of a real engineering department: vertical delegation, horizontal consultation, conflict resolution, and domain isolation, all enforced via hooks and verification gates.

Process enforcement

Collaboration protocol & process gates

Every agent follows a five-step interaction protocol. No action is taken without explicit user approval.

01
Ask
Clarify intent before proposing solutions
02
Options
Present 2–4 alternatives with trade-offs
03
Decide
User makes the call
04
Draft
Agent shows work before committing
05
Approve
Nothing written without explicit sign-off

Implicit workflow commands in CLAUDE.md act as mandatory process checkpoints.

/spec
Blueprint + approval required before any file is created
/plan
Atomic task breakdown before implementation begins
/tdd
Red → Green → Refactor cycle; terminal log required
/context
Diagnose context state; recall from Supermemory first
/annotate
Persist API gotchas, caveats, and workarounds immediately
Verifiable plan format (Rule 12)
[Step] → verify: [specific, testable criterion]
"looks good" and "should work" are not accepted criteria.

Memory architecture

5-layer persistent memory

Files persist across sessions, compactions, and crashes. Incremental loading with a 3-Question Relevance Gate: max 3 Tier 2 files per session, stop if context < 30%.

T1
MEMORY.md
50-line index, keyword triggers — always in context via CLAUDE.md
T2
.claude/memory/*.md
Topic files: annotations, tech decisions, role context — max 3/session, line-range reads preferred
T3
.claude/memory/archive/
Cold storage: sessions, decisions, dreams — grep-search only, never proactive load
T4
MCP Supermemory
Semantic recall across sessions (external). mcp_supermemory_recall before research; mcp_supermemory_memory after major features
T5
CLAUDE.md @include chain
Static universal context — always injected into system prompt, not loaded on demand
